01 · Problem

Manual handoff
wastes everyone's time.

Every token change required copying hex values into Slack, waiting for a developer, hoping nothing breaks. No history, no review, constant errors.

Manual sync

Designers copy values into Slack. Developers retype them into code. Errors happen constantly.

No change history

Nobody knows what changed, when, or why. Bugs traced back to "someone updated a color."

Delayed updates

Token changes sit in Slack for days before a dev has time to implement them.

Diverged sources of truth

Figma says one thing. The codebase says another. Nobody is sure which is correct.

02 · Solution

One button.
Figma to GitHub.

A Figma plugin reads all Variables, compares them to the last pushed state, shows a diff, and creates a Pull Request with an auto-generated change table.

01
Designer updates tokens in Figma Variables

Colors, spacing, typography — any variable type is supported across all collections and modes.

02
Opens plugin → scans tokens

Plugin reads all Variables via Figma Plugin API and shows a visual diff — what was added, what changed, what color it was before.

03
One click — Pull Request created

Plugin pushes design-tokens.json to a new branch and opens a PR with an auto-generated change table in the description.

04
Developer reviews and merges

PR shows exactly what changed. One click merge — tokens are live in production across CSS, SCSS, and TypeScript.
