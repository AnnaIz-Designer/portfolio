04 · Process

How the decision
was made.

01

Research — emotion and color in health contexts

Analyzed how color affects perceived trust, calm, and adherence in health and wellness apps. High-saturation UI increases cognitive load in daily-use tools.

02

Hypothesis — calm palette reduces churn

Proposed that shifting to muted, nature-inspired tones would reduce visual fatigue and improve long-term engagement with the app.

03

New token system built in Figma Variables

Rebuilt the entire color system using Figma Variables with light/dark mode support. All tokens named semantically — surface, text, accent, status.

04

A/B test — new vs old palette

Ran a controlled A/B test with equal user segments over 30 days. Measured retention, session frequency, and subscription conversion.

05

Full rollout based on results

With 78% vs 43% retention, the decision to roll out the new palette was clear. Full implementation followed with automated token sync to the codebase.

05 · Takeaways

What this
project taught.

Color is a product decision, not just a visual one

The palette shift directly influenced user behavior — retention and monetization, not just aesthetics

Semantic token architecture enables faster iteration

A well-structured Variables system let us test and roll out the new palette without touching individual components

Context matters more than aesthetics

A palette that works for a fintech or gaming app can be completely wrong for daily health monitoring
